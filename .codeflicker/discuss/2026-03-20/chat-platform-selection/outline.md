# 讨论：儿童聊天工具平台选择

> 状态：已决策 | 轮次：R3 | 日期：2026-03-20

## 🔵 当前焦点

- **已达成决策**：推荐使用微信小程序作为儿童聊天工具平台
- **下一步**：进入技术方案设计和实施规划阶段

## ⚪ 待讨论

- [ ] 微信小程序具体技术方案设计
- [ ] 后端架构调整（SQLite → 云数据库）
- [ ] 音频存储方案（本地文件系统 → 云存储）
- [ ] AI 对话集成方案（扣子 Bot API vs 直接调用 OpenAI）

## ✅ 已确认

- **使用设备**：主要用手机 (#R2)
- **使用场景**：移动场景，非固定场景 (#R2)
- **对话模式**：接受"说话-等待-回复"模式 (#R2)
- **开发能力**：团队不熟悉具体技术栈，会使用 AI coding (#R2)
- **当前方案定位**：React Web App 用于功能验证 (#R2)
- **微信生态触达**：家长已使用微信，希望通过微信生态触达 (#R3)
- **离线能力**：可有可无，非必须 (#R3)
- **多平台支持**：需要同时支持 iOS 和 Android (#R3)
- **开发者账号**：已有微信企业认证开发者账号 (#R3)
- **平台选择决策**：微信小程序 → [D01-wechat-miniprogram](./decisions/D01-wechat-miniprogram.md) (#R3)

## ❌ 已否决

- React Native App：开发成本高，用户下载门槛高 (#R3)
- PWA：iOS 支持不完善，语音能力受限 (#R3)
- 扣子 SDK (H5)：移动端浏览器权限复杂，不适合儿童使用 (#R3)
- 豆包 App 渠道：界面不可定制，无法注入自有素材库 (#R3)

| 问题 | 结论 | 详情 |
|------|------|------|

| 实时语音流需求 | 不需要，接受说话-等待-回复模式 | #R2 确认 |
