# 决策：选择微信小程序作为儿童聊天工具平台

**决策时间**：#R3
**状态**：✅ 已确认
**关联大纲**：[返回大纲](../outline.md)

---

## 📋 背景

### 问题/需求
根据 plan.md 第二章儿童聊天工具需求，需要选择合适的技术平台，使得：
1. 儿童可以方便地通过语音与智能体连续对话
2. 适合 0-6 岁儿童日常使用
3. 支持移动场景（外出、网络不稳定）
4. 可以通过家长已有的触达渠道传播

### 约束条件
- **用户设备**：主要使用手机
- **使用场景**：移动场景，非固定场景
- **交互模式**：接受"说话-等待-回复"模式，不需要实时语音流
- **开发能力**：团队不熟悉具体技术栈，依赖 AI coding 工具
- **当前方案**：React Web App 用于功能验证，不适合儿童直接使用
- **微信生态**：家长已使用微信，希望通过微信触达
- **多平台需求**：需要同时支持 iOS 和 Android
- **开发者资源**：已有微信企业认证开发者账号

---

## 🎯 目标

选择一个平台，能够：
1. **儿童使用便利**：点击即用，无需复杂操作
2. **开发效率高**：AI coding 友好，快速迭代
3. **移动场景适配**：支持语音交互、网络不稳定应对
4. **触达成本低**：利用已有微信生态，无需额外获客

### 详细对比

#### 微信小程序（✅ 选定）
**优势**：
- 无需下载，家长已有微信，儿童点击即用
- 可添加到手机桌面，形成独立入口
- 原生语音 API（wx.getRecorderManager），权限管理简化
- AI coding 友好（Taro、uni-app 框架支持）
- 天然跨平台（iOS + Android）
- 可嵌入公众号、分享给家长群
- 支持本地存储，可缓存课程音频

**劣势**：
- 包体积限制（主包 2MB），音频需云端存储
- 需要微信开发者账号（已有）
- 教育类小程序审核相对严格

#### React Native App（❌ 否决）
**优势**：
- 完全自定义界面
- 可以调用原生语音能力
- 支持离线功能

**劣势**：
- 需要用户下载 App，儿童操作门槛高
- 开发和调试成本高
- 需要应用商店审核
- AI coding 支持较弱

#### PWA（❌ 否决）
**优势**：
- 无需下载
- 可以添加到桌面

**劣势**：
- iOS 支持不完善
- 浏览器麦克风权限处理复杂
- 儿童使用难度高

#### 扣子 SDK (H5)（❌ 否决）
**优势**：
- 快速集成，1 周上线

**劣势**：
- 移动端浏览器麦克风权限复杂
- 网络不稳定时体验差
- 界面定制能力有限

#### 豆包 App 渠道（❌ 否决）
**优势**：
- 零代码，0.5 天上线
- 内置语音能力

**劣势**：
- 界面完全不可定制
- 无法注入自有素材库
- 无法与现有后端打通

---

## ✅ 最终决策

### 选定方案
**微信小程序**

### 决策理由
1. **儿童使用便利性最高**：
   - 家长可以把小程序添加到桌面，儿童点击图标即用
   - 微信生态内，无需解释"浏览器"概念
   - 语音权限一次授权，后续无需重复确认

2. **AI Coding 友好度高**：
   - Taro / uni-app 框架支持，AI 工具训练数据充足
   - 教育类小程序开源案例多，可参考性强

3. **移动场景适配好**：
   - 原生语音 API，无需依赖浏览器权限
   - 支持本地存储，可缓存已下载音频
   - 天然跨平台

4. **触达成本低**：
   - 家长已使用微信，无需额外获客
   - 可分享到家长群、朋友圈

5. **开发资源已就绪**：
   - 已有微信企业认证开发者账号
   - 无需额外申请

### 预期效果
- 开发周期：2-3 周（AI coding 辅助）
- 儿童使用门槛：低（点击图标即可）
- 多平台支持：天然支持 iOS + Android

---

## ❌ 被否决的方案

### React Native App
- **否决原因**：开发成本高，用户下载门槛高，AI coding 支持较弱
- **重新考虑条件**：如需完全离线功能、不依赖微信生态、更强性能

### PWA
- **否决原因**：iOS 支持不完善，浏览器麦克风权限处理复杂
- **重新考虑条件**：如需快速验证，且不考虑 iOS 用户

### 扣子 SDK (H5)
- **否决原因**：移动端浏览器权限复杂，不适合儿童使用
- **重新考虑条件**：如仅需快速验证对话功能，不作为正式产品

### 豆包 App 渠道
- **否决原因**：界面不可定制，无法注入自有素材库
- **重新考虑条件**：如仅需验证产品逻辑，不考虑品牌和素材管理
